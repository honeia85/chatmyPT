## 한 줄 결론

**자기개선 루프는 "외부 검증 신호에 앵커링된 축적"일 때만 작동한다.** 순수 자기반성은
실증적으로 기각됐고(성능 악화), 자가생성 스킬의 무검증 자동 축적도 기각됐다(평균 이득 없음).
작동이 실증된 것은 ① 실행 검증이 있는 코드 스킬 축적, ② 궤적→자연어 교훈 추출,
③ 외부 신호 기반 반성 — 셋 모두 사람 검토 게이트와 결합할 때다.

## 1. 검증된 핵심 결과 (신뢰도·투표 표기)

### 작동하는 것

| 발견 | 근거 | 신뢰도 |
|------|------|--------|
| **코드 스킬 라이브러리 축적** (Voyager, TMLR 2024): 실행 가능한 코드로 스킬 저장 + 성공 시에만 라이브러리 추가. 고유 아이템 3.3배 등 | [arXiv:2305.16291](https://arxiv.org/abs/2305.16291) | high (3-0) — 단, 결정적 실행 검증이 있는 환경 조건부 |
| **궤적→자연어 교훈 추출** (ExpeL, AAAI 2024): 가중치 업데이트 없이 경험에서 insights를 추출해 인컨텍스트로 회상. "마크다운 위키에 교훈 축적" 패턴의 직접적 학술 근거 | [arXiv:2308.10144](https://arxiv.org/abs/2308.10144) | high (3-0) |
| **스킬 생명주기 관리** (MUSE-Autoskill, 2026): 스킬 = 수명 있는 테스트 가능한 자산. 생성-메모리-관리-평가-정련 5단계 + 스킬별 사용 이력 메모리 + 임계값 기반 자동 은퇴 | [arXiv:2605.27366](https://arxiv.org/html/2605.27366) | medium (프리프린트) |
| **외부 신호 기반 반성** (Reflexion, NeurIPS 2023): 반성이 코드 실행기·테스트 같은 외부 피드백에 앵커링되면 잘 작동 (HumanEval 91%). TACL 2024 서베이도 동일 조건 확인 | [arXiv:2303.11366](https://arxiv.org/abs/2303.11366), [arXiv:2406.01297](https://arxiv.org/abs/2406.01297) | high (3-0) |
| **큐레이션된 스킬의 효과** (SkillsBench, 2026): 사람이 큐레이션한 스킬은 평균 **+16.2pp** 성공률 향상 (도메인 분산 큼: +4.5~+51.9pp) | [arXiv:2602.12670](https://arxiv.org/abs/2602.12670) | high (3-0) |

### 작동하지 않는 것 (실증 기각)

| 발견 | 근거 | 신뢰도 |
|------|------|--------|
| **자가생성 스킬의 자동 축적**: 에이전트가 스스로 만든 스킬은 평균 **−1.3pp** (테스트 구성 중 1개만 개선). 큐레이션 +16.2pp와의 격차가 핵심 | SkillsBench/SoK ([arXiv:2602.20867](https://arxiv.org/html/2602.20867v1)) | high (3-0) |
| **외부 피드백 없는 내재적 자기수정**: GPT-4 GSM8K 95.5% → 반성 2라운드 후 **89.0%로 하락** (ICLR 2024). 초기 성공 보고 대부분은 오라클 정답 라벨로 수정 중단 시점을 정한 방법론 결함 (TACL 2024) | [ICLR 2024](https://openreview.net/forum?id=IkmD3fKBPQ), [arXiv:2406.01297](https://arxiv.org/abs/2406.01297) | high (3-0) |
| **반성의 눈덩이 실패**: 자유 반성은 오류를 고치는 대신 재귀적으로 정당화(hallucination snowballing); 반대로 반성 출력에 과도한 스키마를 강제하면 포맷 함정에 빠짐(structure snowballing, 96/100건 포맷 오귀인) | [arXiv:2604.06066](https://arxiv.org/html/2604.06066v1) | medium (단일 8B 모델) |
| **"축적량 증가 = 성능 향상" 가정**: ExpeL의 경험 풀 단조 증가→성능 향상 주장은 검증에서 **기각** (1-2). 무한정 위키 축적이 자동으로 이득이라는 근거 없음 | 검증 라운드 | refuted |
| **스킬 유통 채널의 공급망 공격**: ClawHavoc(2026-02) — 악성 스킬 ~1,200개가 마켓플레이스 침투, 자격증명 대량 탈취. 감사 없는 외부 스킬 설치 금지의 실증 근거 | SoK + 다수 보안 벤더 교차 확인 | high (3-0) |

### 커버리지 공백 (정직한 한계)

- **축 3 (구조 자기개선: ADAS, Darwin Gödel Machine, SICA, AlphaEvolve)**: 검색은 됐으나
  (DGM: SWE-bench 20→50% 보고, objective hacking 사례 포함) **3표 적대적 검증을 통과한
  주장이 없어 본 보고서에 미포함**. "증거 없음"이 아니라 "검증 통과 주장 없음".
- MemGPT/Letta 등 메모리 아키텍처 세부도 동일. 2026 최신 결과 다수가 비동료심사 프리프린트.
- intrinsic self-correction 실패는 2023~24년 모델 기준 — RL 훈련된 자기수정(SCoRe, o1류)은
  별도 조건으로 성립하며, 2026 프런티어 모델에서 격차가 좁혀졌을 수 있음.

## 2. 우리 워크스페이스 적용 설계 (연구 근거 1:1 매핑)

### 차용할 메커니즘

1. **ExpeL 패턴 — 회고→교훈 후보 축적**: Loop 종료 시(APPROVE/max-rounds) 리뷰 이력
   전체를 입력으로 회고 라운드를 1회 실행, 자연어 교훈을 `wiki/lessons/`에
   **candidate 상태로** 기록. 작업 시작 시 관련 교훈+유사 과거 라운드를 함께 회상.
2. **MUSE 패턴 — 스킬 생명주기**: 각 스킬에 사용 이력(성공/REVISE 사유) 기록을 붙이고,
   체크 가능한 수용 기준(예: paper-reading이면 템플릿 필드 완전성)으로 평가.
   반복 실패 스킬은 은퇴 후보로 표시만 (자동 삭제 금지).
3. **Reflexion 패턴 — 외부 신호 앵커링**: REVISE 판정은 반드시 체크 가능한 근거
   (실행 결과, 원자료 대조, 인용 검증)에 기반. 우리 writing-reviewer가 이미 이 방식
   (round-3에서 CSV 재대조로 순환성을 잡은 것이 정확히 이 패턴).

### 하지 말아야 할 것 (실패 조건 실증 근거)

1. ~~자가생성 스킬 자동 활성화~~ → 에이전트가 만든 스킬 수정은 **diff 초안까지만** (−1.3pp 근거)
2. ~~외부 신호 없는 반성-재시도 무한 루프~~ → REVISE는 체크 가능 기준 필수 + 라운드 상한 (95.5→89.0% 근거)
3. ~~리뷰어가 정답을 안다고 가정하는 평가~~ (오라클 라벨 함정)
4. ~~회고 출력에 과도한 스키마 강제~~ (structure snowballing)
5. ~~감사 없는 외부 스킬 설치~~ (ClawHavoc)

### 사람 검토 게이트 4개

| 게이트 | 대상 | 근거 |
|--------|------|------|
| G1 | 스킬 초안→활성 승격, 기존 스킬 수정 | 큐레이션 +16.2pp vs 자가생성 −1.3pp |
| G2 | CLAUDE.md 규칙 변경 전부 | 규칙은 모든 후속 동작을 조건화 — 최고 위험 지점 (drift 방지) |
| G3 | 위키 교훈 candidate→confirmed 승격 (주기 일괄 검토) | 축적량 단조 이득 가정 기각 |
| G4 | 외부 스킬/플러그인 도입 (코드 감사) | ClawHavoc |

**자동 허용 구간** (게이트 불필요): 교훈 후보 기록, 스킬 사용 통계 축적, 실패 사례 로깅 —
읽기 전용 축적은 위험이 없음.
