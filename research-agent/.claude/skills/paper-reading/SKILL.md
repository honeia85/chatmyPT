# 논문 읽기 Skill

## 입력

- 논문 파일 경로 (`papers/` 아래) 또는 URL
- (선택) 관심 관점 — 예: "leakage 관점에서", "베이스라인 비교 관점에서"

## 출력

1. `wiki/<논문-슬러그>.md` — 요약 페이지 (paper-summary 템플릿)
2. `wiki/<논문-슬러그>-critical.md` — 비판적 읽기 노트 (critical-reading 템플릿)
3. `wiki/literature-matrix.md` 갱신 — 새 행 추가

## 절차

1. `wiki/`에 같은 논문 페이지가 이미 있는지 확인한다. 있으면 새로 만들지 말고 갱신한다.
2. 논문을 읽고 `wiki/templates/paper-summary.md` 형식으로 요약한다:
   문제 / 방법 / 데이터 / 결과 / 한계 5개 섹션을 모두 채운다.
   원문에 없는 내용은 비워두고 "논문에 명시 없음"으로 표시한다.
3. `wiki/templates/critical-reading.md` 형식으로 비판적 읽기 노트를 작성한다:
   주장 / 근거 / 약점 / 재현 가능성.
4. `wiki/literature-matrix.md`에 한 행을 추가한다 (없으면 템플릿으로 생성).
5. 요약 페이지 하단에 관련 위키 페이지 링크를 단다 (같은 주제의 기존 페이지 검색 후).

## 품질 기준

- 결과 섹션에는 논문의 수치를 지표·데이터셋과 함께 그대로 옮긴다 (반올림 재해석 금지).
- 한계 섹션은 저자가 인정한 한계와 내가 발견한 한계를 구분해 적는다.
- 비판적 읽기의 각 약점에는 논문의 해당 섹션/표 번호를 병기한다.
