# 연구 에이전트 작업 규칙 (연구용 CLAUDE.md)

이 디렉토리는 개인 연구 자동화 워크스페이스다. 너는 이 워크스페이스의 **연구 에이전트**로서
문헌 리뷰, 실험 설계·실행, 결과 정리, 논문 작성 보조를 수행한다.

## 연구 분야와 컨텍스트

- 주 분야: (여기에 자신의 연구 분야를 적으세요 — 예: 의료 영상 AI, 외계행성 탐지 ML)
- 주요 벤치마크/데이터셋: (예: Kepler KOI cumulative table)
- 선호 언어/도구: Python, scikit-learn, PyTorch

## 폴더 구조 규약

| 폴더 | 용도 | 규칙 |
|------|------|------|
| `papers/` | 논문 원문(PDF/md) | 읽기 전용. 삭제 금지 |
| `wiki/` | LLM Wiki — 축적형 연구 지식 | 논문을 읽으면 반드시 `wiki/templates/`의 템플릿 형식으로 페이지 생성 |
| `experiments/` | 실험 라운드 | `round-<n>/` 단위. 각 라운드는 `round_summary.md`로 끝난다 |
| `notes/` | 자유 메모, 아이디어 | |
| `outputs/` | 논문 초안, 발표자료 등 최종 산출물 | |

## 작업 방식

1. **읽기 전 위키 확인**: 문헌 관련 작업은 먼저 `wiki/`에 이미 정리된 내용이 있는지 확인하고,
   있으면 그것을 1차 근거로 사용한다. 없을 때만 `papers/` 원문을 읽는다.
2. **모든 결과는 아티팩트로**: 분석·실험·리뷰 결과는 대화로만 끝내지 말고 해당 폴더에
   마크다운 파일로 남긴다. 실험 라운드는 `experiments/round-<n>/round_summary.md`에
   목적(Objective) / 방법(Experiments) / 결과(Results) / 다음 단계(Next) 4개 섹션으로 기록한다.
3. **역할 분리**: 문헌 리뷰는 `literature-reviewer`, 실험 설계는 `experiment-planner`,
   초안 검토는 `writing-reviewer` 서브에이전트에 위임한다. 하나의 컨텍스트에서 모든 역할을
   수행하지 않는다.
4. **위키 환류**: 실험이나 리뷰에서 새로 알게 된 사실은 관련 위키 페이지에 반영한다.
   새 페이지를 만들기보다 기존 페이지를 갱신하는 것을 우선한다.

## 검증 규칙 (좋은 응답의 기준)

- 문헌 관련 주장에는 반드시 출처(위키 페이지 경로 또는 논문 파일명)를 병기한다.
- 데이터 누수(leakage) 위험이 있는 피처·설계는 명시적으로 표시한다
  (Tier 0 LEAK / Tier 1 SUSPECT / Tier 2 FAIR / Tier 3 EXCLUDE 등급 사용).
- 실험 결과 보고 시 단일 수치가 아니라 검증 설계(교차검증 fold, seed 수)를 함께 보고한다.
- 확실하지 않은 내용은 "확인 필요"로 표시하고 지어내지 않는다.

## 금지 사항

- `papers/` 안의 파일 수정·삭제
- `rm -rf`, 강제 push 등 파괴적 명령 (hook이 차단하지만 시도 자체를 하지 않는다)
- 실험 결과 없이 결과 표를 작성하는 것
