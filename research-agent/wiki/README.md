# LLM Wiki — 검색 가능한 연구 기억

일회성 요약이 아니라 **축적형 지식**을 만드는 공간이다. 에이전트와 사람이 함께 읽고 쓴다.

## 운영 규칙

1. **논문 1편 = 페이지 2개**: 요약(`<슬러그>.md`) + 비판적 읽기(`<슬러그>-critical.md`).
   `paper-reading` 스킬이 이 규칙대로 생성한다.
2. **새 논문 유입 시**: `papers/`에 파일을 넣고 "논문 읽어줘"라고 요청하면
   위키 페이지 생성 + `literature-matrix.md`에 행 추가까지 자동으로 진행된다.
3. **갱신 우선**: 같은 주제의 페이지가 이미 있으면 새 페이지를 만들지 않고 기존 페이지를
   갱신한다. 갱신 이력은 페이지 하단 `## 변경 이력`에 날짜와 함께 남긴다.
4. **충돌 해소**: 실험 결과가 위키 서술과 충돌하면 위키를 실험 결과 쪽으로 갱신하고
   근거(라운드 경로)를 남긴다.
5. **연구 질문/공백**: 여러 논문을 연결해 발견한 연구 공백은 `research-gaps.md`에 모은다.

## 파일 구성

- `templates/paper-summary.md` — 논문 요약 템플릿 (문제/방법/데이터/결과/한계)
- `templates/critical-reading.md` — 비판적 읽기 템플릿 (주장/근거/약점/재현성)
- `templates/literature-matrix.md` — 논문 비교 매트릭스 템플릿
- `literature-matrix.md` — 실제 매트릭스 (스킬이 생성·갱신)
- `research-gaps.md` — 연구 질문·공백 모음
