# chatmyPT

Claude API 기반 멀티에이전트 오케스트레이션 시스템

## 특징

- **Claude API** (Sonnet 4.5) 기반 — 200K 컨텍스트, 높은 구조화 출력 신뢰도
- **4단계 계층적 에이전트** — 대기업 조직 구조 (CEO → 본부장 → 팀장 → 연구원)
- **재귀적 업무 위임** — 복잡한 작업을 자동으로 세분화
- **파일 기반 입력** — PDF, CSV, TXT, JSON 등 다양한 파일 분석
- **실시간 조직도** — 에이전트 진행 상태를 실시간 확인
- Gradio 웹 인터페이스

## 시스템 요구사항

- Python 3.8 이상
- Anthropic API 키 (Claude Max 구독 또는 API 계정)

## 설치 방법

```bash
git clone https://github.com/honeia85/chatmyPT.git
cd chatmyPT
pip install -r requirements.txt
```

## API 키 설정

```bash
export ANTHROPIC_API_KEY='your-api-key'
```

모델 변경 (선택):
```bash
export CLAUDE_MODEL='claude-sonnet-4-5-20250929'  # 기본값
export CLAUDE_MAX_TOKENS=1024                      # 기본값
```

## 실행 방법

```bash
python orchestrate.py  # 웹 UI (http://localhost:7861)
```

기본 채팅 (Qwen 로컬 모델):
```bash

```

## 멀티에이전트 오케스트레이터

### 조직도 (4단계)

```
🏢 대표이사 (CEO) ─ 요청 분석, 업무 배정, 최종 종합
├─ 📊 데이터사이언스본부 (본부장)
│  ├─ 📈 통계분석팀 (팀장)
│  │  ├─ 📉 기술통계 연구원
│  │  └─ 🎯 추론통계 연구원
│  └─ 🔍 데이터마이닝팀 (팀장)
│     ├─ 🔎 패턴분석 연구원
│     └─ ⚠️ 이상탐지 연구원
├─ 📝 문서연구본부 (본부장)
│  ├─ 📚 문헌분석팀 (팀장)
│  │  ├─ 💡 핵심분석 연구원
│  │  └─ ⚖️ 비교분석 연구원
│  └─ ✂️ 요약정리팀 (팀장)
│     ├─ 📋 요약 연구원
│     └─ 🗂️ 구조화 연구원
├─ 🔬 과학연구본부 (본부장)
│  ├─ 🧪 가설검증팀 (팀장)
│  │  ├─ 🧠 논리검증 연구원
│  │  └─ 📊 실증분석 연구원
│  └─ ⚗️ 실험설계팀 (팀장)
│     ├─ 📐 방법론 연구원
│     └─ 🎛️ 변수통제 연구원
└─ 📋 전략기획본부 (본부장)
   ├─ 🖊️ 보고서팀 (팀장)
   │  ├─ ✏️ 초안작성 연구원
   │  └─ 📝 편집 연구원
   └─ ✅ 품질관리팀 (팀장)
      ├─ 🔎 검증 연구원
      └─ ⚡ 최적화 연구원
```

### 작동 방식

1. `input/` 폴더에 파일을 넣거나 웹 UI에서 업로드합니다
2. 원하는 분석을 요청합니다
3. **CEO**가 요청을 분석하고 관련 **본부**에 업무를 배정합니다
4. **본부장**이 업무를 파악하고, 복잡하면 **팀장**에게 위임합니다
5. **팀장**이 세부 업무를 **연구원**에게 배정합니다
6. 결과가 역순으로 종합됩니다: 연구원 → 팀장 → 본부장 → CEO → 사용자

### 핵심 설계

- **재귀적 위임**: `_run_unit()` 메서드가 모든 계층에서 동일하게 동작
- **동적 에이전트 생성**: 요청에 따라 필요한 조직만 활성화
- **단일 API 클라이언트 공유**: 모든 에이전트가 하나의 API 연결 공유
- **조직 구조 = 설정**: `departments.py`만 수정하면 조직 변경 가능

### 지원 파일 형식

| 형식 | 확장자 | 처리 방식 |
|------|--------|----------|
| 텍스트 | .txt, .md, .log | 직접 읽기 (10,000자) |
| CSV | .csv, .tsv | 헤더 + 상위 100행 |
| PDF | .pdf | PyPDF2 텍스트 추출 |
| JSON | .json | 구조화된 텍스트 변환 |
